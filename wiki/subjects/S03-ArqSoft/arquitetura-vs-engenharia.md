# Arquitetura de Software vs Engenharia de Software

Arquitetura de software é um **subcampo** da engenharia de software. São áreas complementares, com focos distintos.

## Engenharia de Software
Foca no **processo de desenvolvimento**. Trata de todos os aspectos sobre como o software é produzido — ferramentas, metodologias, produtividade, qualidade do processo, ciclo de vida. Estuda, desenvolve e aplica ferramentas para ganhar produtividade e qualidade.

## Arquitetura de Software
Foca no **projeto (design) do sistema**. Trata dos projetos dos sistemas de software e da maneira como os componentes desses sistemas se comunicam entre si. Define a estrutura, os padrões e as decisões de alto nível.

## Resumo
| Dimensão | Engenharia | Arquitetura |
|----------|------------|-------------|
| Foco | Processo | Projeto (design) |
| Pergunta central | Como desenvolvemos software? | Como o sistema é estruturado? |
| Escopo | Ciclo de desenvolvimento | Sistema e seus componentes |

## Papel do arquiteto vs papel do engenheiro
- **Arquiteto de software:** papel **consultivo e orientativo** — apoia decisões de design, padrões, tradeoffs, e guia as equipes técnicas.
- **Engenheiro de software:** papel **mais gerencial e orientado ao código** — executa, implementa e gerencia o trabalho de construção.

## Escopo da arquitetura de software
Envolve:
- Decidir padrões arquiteturais (ou adaptações) a utilizar
- Modelar diagramas para representar visão geral do sistema
- Especificar requisitos não-funcionais (desempenho, segurança, escalabilidade)
- Definir responsabilidades e funcionalidades de cada componente
- Considerar ciclo de vida do sistema (correções, evoluções)
- Estudar e consolidar padrões e boas práticas
- Analisar pontos críticos de falha e tradeoffs
- Manter documentação detalhada do sistema

**Não faz parte do escopo:** realizar testes automatizados (essa é responsabilidade de engenharia/qualidade de software, não de arquitetura).
